## üèóÔ∏è Architecture Summary

### The Fractal Model

```
Scale 0: Mathematics (Universal patterns)
    ‚Üì Proven by theorems
Scale 1: Libraries (@noble/*, graphlib, etc.)
    ‚Üì Audited implementations
Scale 2: MCP Servers (Network services)
    ‚Üì Protocol-wrapped
Scale 3: Embedded Patterns (Agent code)
    ‚Üì Domain-specific
Scale 4: Agent Operations (Application)
    ‚Üì Business logic
Adaptive: Pattern Resolver chooses optimal scale
```

### The Three Integration Models

**Model A: Embedded** (Monolithic)
- Agents use embedded patterns directly
- Simple deployment (single process)
- Low latency (function calls)
- Use when: Single-node, edge, CLI

**Model B: MCP Fabric** (Microservices)
- Agents call MCP servers
- Distributed deployment (multi-process)
- Reusable primitives (shared services)
- Use when: Multi-region, shared infrastructure

**Model C: Adaptive** (Hybrid)
- Pattern resolver chooses based on context
- Flexible (works in any environment)
- Optimal (uses best available)
- Use when: Uncertain or evolving deployment

**v3.1 Achievement**: All three models now supported (was Model A only)

## üìö Glossary

**Fractal Architecture**: Patterns recurring at multiple scales (math ‚Üí code ‚Üí services ‚Üí apps)

**MCP Fabric**: Layer of MCP servers providing reusable primitives (not P2P mesh)

**Adaptive Resolution**: Context-aware selection of pattern implementation (MCP vs embedded vs library)

**Episodic Memory**: Time-indexed experiences (concrete events)

**Semantic Memory**: Abstract knowledge (facts independent of time)

**Convergent Aggregation**: Merging that converges to stable value (Pattern #6)

**Byzantine Resistance**: Tolerates < 1/3 malicious nodes (Pattern #8)

**Gossip Protocol**: Epidemic spreading, O(log N) convergence (Pattern #4) - designed for v3.2

**CRDT**: Conflict-Free Replicated Data Type (Pattern #10) - designed for v3.4
